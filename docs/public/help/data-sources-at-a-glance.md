﻿﻿# Data Sources at a Glance (Public)

## Purpose
Provide a simple map of the kinds of inputs Alpha Engine uses and how they flow to outputs.

## Audience
- End users
- Evaluators

## When to use this
- You want to understand what kinds of data are involved without internal integration detail.

## Prereqs
- None

---

## Input categories (current repo configuration)
Configured sources live in `config/sources.yaml` and are validated by `app/ingest/validator.py`.

### News
- `alpaca_news_main` (adapter: `alpaca_news`) — symbols: NVDA/AAPL/AMD

### Market / market structure
- `yahoo_market_watch` (adapter: `yahoo_finance`) — symbols: SPY/QQQ
- `etf_flows_market` (adapter: `etf_flows`) — symbols: SPY/QQQ/IWM
- `options_sentiment` (adapter: `options_flow`) — symbols: SPY/TSLA/NVDA

### Macro / regime
- `fred_rates` (adapter: `fred_macro`) — series: FEDFUNDS
- `yield_curve_spread` (adapter: `fred_macro`)
- `fear_greed_index` (adapter: `fear_greed`)
- `cross_asset_core` (adapter: `cross_asset`)
- `market_breadth` (adapter: `market_breadth`)
- `market_baseline` (adapter: `market_baseline`)

### Social / crowd
- `reddit_wsb` (adapter: `reddit_social`) — subreddit: wallstreetbets
- `google_trends_main` (adapter: `google_trends`) — keywords: bitcoin/recession/bull market

### Volatility events / calendar
- `earnings_tracker` (adapter: `earnings_calendar`)

### Bundles (local)
- `developer_bundle_ai_supply` (adapter: `custom_bundle`) — local file: `bundles/ai_supply.json`

## Output artifacts (what you can actually inspect)

### Demo exports (fastest)
Generated by `python scripts/demo_run.py`:
- `outputs/scored_events.csv`
- `outputs/mra_outcomes.csv`
- `outputs/predictions.csv`
- `outputs/strategy_performance.csv`

### SQLite evidence trail (historical backfill + replay)
Generated by `python -m app.ingest.backfill_cli backfill-range ...`:
- DB: `data/alpha.db`
- Tables include: `raw_events`, `scored_events`, `mra_outcomes`, `predictions`, `prediction_outcomes`, `signals`, `consensus_signals`
