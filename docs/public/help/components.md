## Ingestion

### What it does
Fetches enabled sources, normalizes fields, validates, dedupes, persists to SQLite, and routes events into tracks.

### Inputs/outputs
- Inputs:
  - `config/sources.yaml` (validated by `app/ingest/validator.py`)
  - Provider keys via `config/keys.yaml` + environment variables (resolved by `app/ingest/key_manager.py`)
- Outputs:
  - SQLite `data/alpha.db` (ingestion tables written by `app/ingest/event_store.py`, notably `events`)
  - Track routing buckets (sentiment/quant/regime/crowd/alpha) via `app/ingest/router.py`

### Key knobs (safe)
- Enable/disable sources (per environment)
- Poll intervals and backfill windows (as configured)

### Common issues + fixes
- Rate limits/timeouts: increase poll interval; check provider mapping (`app/ingest/runner_core.py`).
- Missing events: check timestamp extraction and validator drops (`app/ingest/validator.py`).

### Where to read next
- Operator guide: `ADMIN_GUIDE.md`
- Ops ingestion checks: `docs/internal/ops/ingestion-health-checks.md`

## Prediction engine (high level)

### What it does
Turns scored events + market context into predictions, evaluates outcomes, and supports ranking/learning loops in fuller paths.

### Inputs/outputs
- Inputs:
  - Scoring: `app/core/scoring.py`
  - MRA: `app/core/mra.py`
  - Context/features: derived during backfill/replay and passed to runners
- Outputs:
  - Demo exports: `outputs/` (via `python scripts/demo_run.py`)
  - Replay evidence trail (including outcomes): `data/alpha.db` written by `app/ingest/replay_engine.py`

### Consensus weighting (why signals differ)
- Consensus engine (demo path): `app/runtime/consensus.py`
- Consensus includes weights (`ws`, `wq`) and agreement metadata for debugging.

### Where to read next
- System design (internal): `docs/archive/SYSTEM_DESIGN.md`
- Limitations (internal): `docs/internal/audit/model-limitations.md`

## UI (Streamlit)

### What it does
Renders dashboard, intelligence, and audit views against `data/alpha.db`.

### Entry points
- Unified shell: `app/ui/app.py` → `python -m streamlit run app/ui/app.py`
- Audit page: `app/ui/audit.py` → `python -m streamlit run app/ui/audit.py`

### Key knobs (safe)
- `ALPHA_DB_PATH` (defaults to `data/alpha.db`)

### Common issues + fixes
- Wrong DB: set `ALPHA_DB_PATH` and open audit view to confirm expected tables exist.
