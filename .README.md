# Alpha Engine POC — CLI Reference

This repo has a few “operator” entrypoints that are meant to be run from the repo root (the same folder as `app/`).

## Backfill CLI (market/news ingest)

Entry: `python -m app.ingest.backfill_cli <command> [args]`

Commands:

- `run --days 90`  
  Run the full backfill for the last N days (default `90`).

- `backfill-range --start YYYY-MM-DD --end YYYY-MM-DD [--no-replay] [--no-fail-fast] [--max-zero-insert-slices N]`  
  Backfill a specific time range.
  - `--replay/--no-replay`: replay pipeline after fetch (default replay)
  - `--skip-completed/--no-skip-completed`: skip refetching slices already completed per source (default skip)
  - `--fail-fast/--no-fail-fast`: abort on suspicious “fetched but inserted 0” slices (default true)
  - `--max-zero-insert-slices`: tolerated consecutive zero-insert slices (default `2`)

- `list-target-stocks [--asof YYYY-MM-DD|ISO]`  
  Print the canonical Target Stocks universe (and whether each symbol is active as-of an optional date).

- `add-target-stock SYMBOL [--group NAME] [--active-from YYYY-MM-DD|ISO] [--disabled]`  
  Add/update a ticker in the canonical universe.

- `remove-target-stock SYMBOL`  
  Remove a ticker from the canonical universe.

- `enable-target-stock SYMBOL` / `disable-target-stock SYMBOL`  
  Toggle enabled/disabled without deleting the spec.

Examples:

```bash
python -m app.ingest.backfill_cli run --days 30
python -m app.ingest.backfill_cli backfill-range --start 2024-02-20 --end 2024-03-20
python -m app.ingest.backfill_cli list-target-stocks --asof 2024-03-01
```

## Prediction Sync / Efficiency Rating CLI (scoring + rankings)

Entry: `python -m app.engine.score_predictions_cli <command> [args]`

Commands:

- `eval-window --range YYYY-MM-DD:YYYY-MM-DD [--ingress-range YYYY-MM-DD:YYYY-MM-DD] [--timeframe 1d] [--regime TAG] [--ticker TICKER --ticker TICKER...] [--model directional_drift] [--cap 0.05] [--vol-lookback 20] [--no-materialize-actual]`  
  One-shot end-to-end flow: create a `prediction_run`, build consensus `predicted_series_points`, score vs actuals, and print a within-run ranking.

- `score-predictions --range YYYY-MM-DD:YYYY-MM-DD [--ingress-range YYYY-MM-DD:YYYY-MM-DD] [--timeframe 1d] [--regime TAG] [--ticker T] [--strategy-id ID] [--run-id ID] [--no-materialize-actual] [--db PATH]`  
  Creates (or reuses) a `prediction_run`, aligns predicted vs actual series, computes sync + efficiency metrics, and persists rows into `prediction_scores`.
  - `--range`: the prediction/evaluation window to score (inclusive)
  - `--ingress-range`: optional “context window” label; if omitted defaults to 30 days before prediction start
  - `--materialize-actual/--no-materialize-actual`: if actual points are missing, populate them from `price_bars` (default true)
  - `--regime`: optional tag to support regime-specific efficiency slices (e.g. `LOW|NORMAL|HIGH`)

- `backfill-scores [--limit N] [--no-materialize-actual] [--db PATH]`  
  Finds prediction runs with no scores and scores them.

- `rank-strategies [--ticker T] [--timeframe 1d] [--forecast-days N] [--regime TAG] [--min-samples N] [--min-total-forecast-days N] [--limit N] [--db PATH]`  
  Ranks strategy variants by average `efficiency_rating` (with optional guardrails to avoid small-window bias).

Examples:

```bash
python -m app.engine.score_predictions_cli eval-window --range 2024-03-21:2024-04-21 --ingress-range 2024-02-20:2024-03-20 --timeframe 1d --rank-limit 10
python -m app.engine.score_predictions_cli score-predictions --range 2024-03-21:2024-04-21 --ingress-range 2024-02-20:2024-03-20 --timeframe 1d
python -m app.engine.score_predictions_cli score-predictions --range 2024-03-21:2024-04-21 --timeframe 1d --regime HIGH
python -m app.engine.score_predictions_cli rank-strategies --ticker AAPL --timeframe 1d --min-samples 20 --min-total-forecast-days 200
```
